Data Processing Pipeline

Step 1: IOB2 Parser

def parse_iob2_file(file_path, num_sentences=1000):
    sentences, current = [], []
    with open(file_path, 'r') as f:
        for line in f:
            line = line.strip()
            if not line:
                if current:
                    sentences.append(current)
                    current = []
                continue
            parts = line.split()
            if len(parts) == 2:
                current.append((parts[0], parts[1]))
    return sentences

Tokenization & Label Alignment

Challenge: Subword Tokenization Mismatch

  • XLM-RoBERTa uses SentencePiece tokenizer → splits words into subwords
  • Word "Еуразиялық" → tokens ["▁Еуразия", "лық"] — labels must be aligned
  • Special tokens ([CLS], [SEP]) → assigned label -100 (ignored in loss)
from transformers import AutoTokenizer

tokenizer = AutoTokenizer.from_pretrained("xlm-roberta-base")

def align_tokens_and_labels(sentence, tokenizer):
    words  = [t[0] for t in sentence]
    labels = [t[1] for t in sentence]
    enc = tokenizer(words, is_split_into_words=True, truncation=True)
    word_ids = enc.word_ids()
    aligned = []
    for wid in word_ids:
        if wid is None:
            aligned.append(-100)  # ignore special tokens
        else:
            aligned.append(tag2id[labels[wid]])
    return enc, aligned

Model Fine-Tuning

XLM-RoBERTa Configuration

from transformers import (
    AutoModelForTokenClassification,
    TrainingArguments, Trainer
)

model = AutoModelForTokenClassification \
  .from_pretrained(
    "xlm-roberta-base",
    num_labels=45,
    id2label=id2tag,
    label2id=tag2id
)

args = TrainingArguments(
  output_dir="./ner-kazakh",
  num_train_epochs=5,
  per_device_train_batch_size=16,
  learning_rate=2e-5,
  weight_decay=0.01,
  evaluation_strategy="epoch",
  save_strategy="epoch",
  load_best_model_at_end=True,
)

Why XLM-RoBERTa?

  • Pre-trained on 100 languages including Kazakh
  • Strong cross-lingual transfer capabilities
  • Proven performance on low-resource NER tasks
  • 278M parameters — good quality/compute balance

Training Details

  • Epochs: 5 (early stopping patience=2)
  • Optimizer: AdamW (lr=2e-5)
  • Batch size: 16
  • Max sequence length: 128 tokens
  • Training time: ~45 min on GPU

API Integration

FastAPI Endpoint

from fastapi import FastAPI
from transformers import pipeline

app = FastAPI()
ner = pipeline(
    "ner",
    model="./ner-kazakh/best",
    tokenizer="xlm-roberta-base",
    aggregation_strategy="simple"
)

@app.post("/predict")
def predict(text: str):
    entities = ner(text)
    return {
      "text": text,
      "entities": entities
    }

Challenges & Solutions

🔴 Subword Alignment

  • Problem: SentencePiece splits Kazakh agglutinative words into many subwords
  • Solution: Implemented word_ids() mapping to propagate labels correctly to subword tokens

🟡 Class Imbalance

  • Problem: O-tags dominate (~80%); rare entities (ADAGE, ART) have few samples
  • Solution: Used weighted loss function & focused evaluation on entity-level F1

🟢 Low-Resource Language

  • Problem: Limited Kazakh NLP tools & pretrained models
  • Solution: Leveraged XLM-RoBERTa's multilingual pretraining for cross-lingual transfer
